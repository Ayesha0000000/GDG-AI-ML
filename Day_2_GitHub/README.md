# Day 2: GitHub Practice

## Objective

To learn Git and GitHub workflow including commits, branches, and pull requests.

---

## Tasks Performed

* Installed Git and checked version
* Configured Git username and email
* Created a repository on GitHub
* Initialized Git in project:
  git init
* Added files:
  git add .
* Committed changes:
  git commit -m "Initial commit"
* Connected project to GitHub:
  git remote add origin <repo_url>
* Pushed code:
  git push -u origin main
* Created a new branch:
  git checkout -b feature/readme
* Created README file and pushed it
* Created Pull Request and merged it

---

## Why This is Important

Git helps us track changes in our project.
GitHub allows us to store code online and collaborate with others.
Branches and PR help in managing work without breaking main code.

---

## What I Learned

I learned how Git works and how to push code to GitHub.
I also learned about branches and resolving merge conflicts.

---

## Result

Successfully created GitHub repository and pushed code with proper workflow.
